# PRD 검증 기준

> ralphi가 PRD 아티팩트를 검증할 때 참조하는 상세 기준.
> harnish M5 이후 고도화 예정. 현재는 최소 기준만 정의.

## 필수 섹션

- §4 구현 명세: 파일 경로, 함수명, 변경 유형 존재
- §6 테스트 기준: bash 명령 또는 조건 목록 존재
- §7 가드레일: hard/soft 구분 존재

## 모호 표현 금지 목록

"적절히", "필요하면", "나중에", "등등", "기타", "충분히", "가능하면"
→ 발견 시 구체적 표현으로 교체 요구

## 섹션 번호 일관성

§1 → §2 → ... → §N 순서. 중간 누락 불가.

## 파일 경로 실존성

§4에 명시된 파일 경로가:
- 이미 존재하거나 (modify/delete)
- create로 명시되거나
- 경로 확인 불가 시 경고
